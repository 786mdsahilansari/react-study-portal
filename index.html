📍 Section A: 1 Mark Questions (Compulsory)

Answer all 12 questions. Each question carries 1 mark. Answers should not be single words or fill-in-the-blanks.

1 Explain the difference between functional and class components in React. 1 Mark

Answer:

Functional Components: JavaScript functions that return JSX. They are simpler, support Hooks, and are preferred in modern React.

Class Components: ES6 classes extending React.Component. They have lifecycle methods and use this.state for state management. Older approach.

2 What is JSX and why is it used in React applications? 1 Mark

Answer:

JSX (JavaScript XML) is a syntax extension that allows writing HTML-like code in JavaScript. It makes code more readable and maintainable by mixing markup with logic. JSX is compiled to React.createElement() calls by Babel.

3 What is the significance of the key prop when rendering lists in React? 1 Mark

Answer:

The key prop helps React identify which items have changed, been added, or removed. Using unique identifiers as keys improves performance and prevents bugs when lists are reordered or filtered. Never use array index as key.

4 How does conditional rendering work in React? Provide one example. 1 Mark

Answer:

Conditional rendering shows or hides elements based on conditions using ternary operators, && operator, or if/else statements.

Example: {isLoggedIn ? <Dashboard /> : <LoginForm />}

5 Explain what props are in React and how they differ from state. 1 Mark

Answer:

Props: Read-only parameters passed from parent to child components. Cannot be modified by child.

State: Local mutable data managed within a component. Can be changed and causes re-render when updated.

6 What are the rules for converting HTML to valid JSX? 1 Mark

Answer:

Key JSX Rules:

  • Use className instead of class
  • Use camelCase for event handlers (onClick)
  • Close all tags, including self-closing tags with />
  • Return single root element (or use Fragment)
  • Use curly braces {} for JavaScript expressions
7 What is component nesting and why is it important in React? 1 Mark

Answer:

Component nesting means placing components inside other components. It allows building complex UIs from smaller, reusable pieces and creates a hierarchical structure. Improves code organization and maintainability.

8 Explain the concept of a pure component and why it matters. 1 Mark

Answer:

A pure component returns the same output for the same input without side effects. It doesn't modify props or external state during render. Pure components are easier to test, understand, and optimize.

9 How do you pass data through multiple levels of components in React? 1 Mark

Answer:

You can pass data through props drilling (passing props through each level) or use Context API to provide data to deeply nested components without passing through every level.

10 What is the difference between default export and named export in React? 1 Mark

Answer:

Default Export: Only one per file, imported as: import Component from './Component'

Named Export: Multiple per file, imported as: import { Component } from './Component'

11 Explain how curly braces are used in JSX for embedding JavaScript expressions. 1 Mark

Answer:

Curly braces {} in JSX allow embedding JavaScript expressions like variables, function calls, ternary operators, and array methods. {expression} is evaluated and its result is rendered. Double braces {{}} are used for inline objects/styles.

12 What is the component tree and how does it relate to prop drilling? 1 Mark

Answer:

Component tree is the hierarchical structure of components. Prop drilling occurs when data flows through many intermediate components that don't use it, just to reach a deeply nested component. Context API helps avoid this.

Answer:

Data Flow in React: React follows unidirectional data flow where data flows only from parent to child via props. Props are read-only; child cannot modify parent data.

Explanation: Parent maintains data and passes to children via props. Children cannot modify props (immutable). Child needs different data, must ask parent to change it.

App

1 What is the difference between handling events in HTML and React? 1 Mark

Answer:

HTML: Event handlers as strings: onclick="handleClick()"

React: Event handlers as functions: onClick={handleClick}. Uses camelCase, passes event object, supports event delegation.

2 Explain the purpose of the useState Hook in React. 1 Mark

Answer:

useState adds state to functional components. Returns an array with current state value and function to update it. Triggers re-render when state changes. Syntax: const [state, setState] = useState(initialValue)

3 What are dependencies in the useEffect Hook and why are they important? 1 Mark

Answer:

Dependencies array tells useEffect when to run. Empty array [] = run once on mount. No array = run every render. [variable] = run when variable changes. Prevents unnecessary effects and controls re-renders.

4 What is event propagation in React? How can you stop it? 1 Mark

Answer:

Event propagation is when events bubble up from child to parent elements. Stop it using e.stopPropagation() in event handler. Prevent default behavior with e.preventDefault().

5 What is the difference between controlled and uncontrolled components? 1 Mark

Answer:

Controlled: Component value controlled by React state. Uncontrolled: Component manages its own state via DOM. Controlled components are preferred for React.

6 Explain the concept of lifting state up in React. 1 Mark

Answer:

When multiple components need same state, move state to their common parent. Parent manages state and passes to children via props. Children communicate via callbacks to update parent state.

7 What are side effects in React? Give examples. 1 Mark

Answer:

Side effects are operations that interact with world outside components: fetching data, updating DOM, subscriptions, timers. useEffect Hook handles side effects. Examples: API calls, document.title changes, event listeners.

8 What is the useReducer Hook and when should it be used instead of useState? 1 Mark

Answer:

useReducer manages complex state with reducer function. Use when state logic is complex, multiple related state values, or next state depends on previous. Better than multiple useState for interconnected state.

9 Explain immutability in state management. Why is it important? 1 Mark

Answer:

Immutability means not modifying original state directly. Create new objects/arrays with changes. Important for React to detect changes, prevent bugs, enable undo/redo, optimize re-renders with shallow comparison.

10 What is state batching and how does it improve performance? 1 Mark

Answer:

State batching groups multiple state updates together in one re-render instead of separate re-renders. React 18+ auto-batches updates. Improves performance by reducing unnecessary DOM updates and re-renders.

11 What is the Context API and how does it solve prop drilling? 1 Mark

Answer:

Context API allows passing data through component tree without passing props manually. Create context with createContext(), use Provider to supply value, consume with useContext() Hook. Solves prop drilling for deeply nested components.

12 Explain the render and commit phases in React's update cycle. 1 Mark

Answer:

Render Phase: React calls component and calculates what to display. Commit Phase: React applies changes to DOM. useEffect runs after commit phase. Understanding this improves performance optimization.

function UserDashboard({ userId }) { const [user, setUser] = React.useState(null); const [posts, setPosts] = React.useState([]); const [loading, setLoading] = React.useState(true); const [error, setError] = React.useState(null); // Effect 1: Fetch user data when userId changes React.useEffect(() => { let isMounted = true; // Prevent memory leak const fetchUser = async () => { try { const response = await fetch(`/api/users/${userId}`); const data = await response.json(); if (isMounted) setUser(data); } catch (err) { if (isMounted) setError(err.message); } }; fetchUser(); // Cleanup function return () => { isMounted = false; }; }, [userId]); // Dependency: userId // Effect 2: Fetch posts when user data available React.useEffect(() => { if (!user) return; const fetchPosts = async () => { try { const response = await fetch(`/api/posts/${user.id}`); const data = await response.json(); setPosts(data); } finally { setLoading(false); } }; fetchPosts(); }, [user]); // Dependency: user object // Effect 3: Set document title React.useEffect(() => { if (user) { document.title = `${user.name}'s Profile`; } return () => { document.title = "Default Title"; }; }, [user]); // Dependency: user if (loading) return <div>Loading...</div>; if (error) return <div>Error: {error}</div>; if (!user) return <div>No user found</div>; return ( <div> <h1>{user.name}</h1> <h2>Posts:</h2> <ul> {posts.map(post => ( <li key={post.id}>{post.title}</li> ))} </ul> </div> ); }

function SearchableUserList() { const [query, setQuery] = React.useState(''); const [results, setResults] = React.useState([]); const [loading, setLoading] = React.useState(false); // Debounce search React.useEffect(() => { const timer = setTimeout(() => { if (query.trim()) { setLoading(true); // Simulate API call fetch(`/api/users?search=${query}`) .then(res => res.json()) .then(data => { setResults(data); setLoading(false); }); } else { setResults([]); } }, 300); // 300ms debounce return () => clearTimeout(timer); // Cleanup }, [query]); return ( <div> <input type="text" value={query} onChange={(e) => setQuery(e.target.value)} placeholder="Search users..." /> {loading && <p>Searching...</p>} <ul> {results.map(user => ( <li key={user.id}>{user.name}</li> ))} </ul> {!loading && results.length === 0 && query && ( <p>No results found</p> )} </div> ); }
